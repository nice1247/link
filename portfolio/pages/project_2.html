1. 프로젝트 명 : 꿈이음 청춘카페 시스템 구축 및 고도화서비스 역할 : 청년 카페에서 사용할 수 있는 마일리지 기반 결제 시스템 및 관리자 플랫폼 참여 기간 : Ver 1.0 (구축): 2023년 07월 ~ 2023년 08월 Ver 2.0 (고도화): 2025년 06월 ~ 2025년 07월 (결제 방식 변경)

2. 프로젝트 전반에 걸친 데이터 모델링을 주도하고, 핵심 결제 API 및 관리자 시스템 백엔드를 개발했습니다. 특히, 운영 중 사용자 증가에 따른 성능 저하 이슈를 해결하고 보안 강화를 위해 결제 로직을 재설계하는 고도화 작업을 담당했습니다.

3. 프레임워크 : eGovFrame (전자정부 표준 프레임워크), Java 데이터베이스 : MySQL (MariaDB) 웹/서버 : Apache, Tomcat, JavaScript, QR 코드 처리 로직 협업/버전 관리 : GitLab

4. - 마일리지 결제 시스템 핵심 로직 구축 (Ver 1.0) [QR 기반 결제 구현] 카페 점주와 사용자 간의 QR 코드 인증을 통한 마일리지 결제 API 시스템을 구현했습니다. 성과: 복잡한 오프라인 결제 과정을 단순화하고, 실시간 마일리지 잔액 확인 및 차감 트랜잭션의 무결성을 확보했습니다. - 사용자 트래픽 대응 및 쿼리 성능 최적화 [쿼리 최적화] 연간 사용자 수 증가(800명 → 1,500명)에 따라 발생하는 정산 및 통계 페이지의 지연 현상을 해결하기 위해 쿼리 최적화 작업을 진행했습니다. 성과: 인덱스 설계 및 대용량 데이터 처리 로직 개선을 통해 정산 통계 페이지의 로딩 시간을 [정량적 수치: 예시: 5초에서 1초로] 단축시키며 운영 안정성을 확보했습니다. - 보안 강화 및 결제 무결성 증대를 위한 시스템 고도화 (Ver 2.0) [결제 방식 고도화] 사용자의 악의적인 접근 및 결제 오용 가능성을 원천 차단하기 위해 결제 인증 방식을 OTP (One-Time Password) 방식으로 전환하는 작업을 수행했습니다. 성과: 매번 고유한 토큰을 사용하는 OTP 로직을 백엔드에 통합함으로써, 기존 결제 방식의 보안 취약점을 해결하고 결제 트랜잭션의 무결성과 속도를 동시에 향상시켰습니다. - 앱 버전 관리를 위한 API 설계 [버전 관리 체계] 모바일 앱 서비스의 특성을 고려하여, 사용자 이탈 방지 및 안정적인 서비스 제공을 위한 앱 버전 관리 API를 설계했습니다. 역할: 구버전 사용자에게는 서비스 이용을 허용하는 동시에, 신규 기능 및 보안 패치가 적용된 버전을 강제/선택 업데이트할 수 있는 유연한 버전 관리 로직을 백엔드에 구축했습니다.

5. - 트래픽 기반의 최적화 경험: 실제 운영 환경에서 사용자 트래픽 증가가 시스템 성능에 미치는 영향을 체감하고, 성능 병목 현상을 쿼리 레벨에서 해결하는 실무 경험을 쌓았습니다. - 보안과 사용성 균형: 마일리지와 같은 금융 관련 기능에서 **보안 무결성(OTP 도입)**과 사용자 편의성 사이의 균형점을 찾는 시스템 설계 능력을 강화했습니다.
