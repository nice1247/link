[세션 끊김 문제 해결] 네이버 앱 등 외부 앱 브라우저 사용 환경에서 간헐적인 세션 끊김 현상이 발생하여 예약이 실패하는 치명적인 문제를 발견했습니다. 해결 : Redis를 도입하여 기존의 Tomcat 세션 대신 중앙 집중식 세션 관리를 수행하도록 구조를 변경했습니다. 성과 : 세션 안정성을 획기적으로 확보하여 모든 브라우저 환경에서 예약 프로세스의 무결성을 보장했습니다. - 예측 기반 요구사항 대비 및 코드 유연성 확보 [요구사항 대처] 짧은 개발 기간 동안 가장 많은 변동 및 피드백을 받은 예약 시스템 개발을 진행하며, 선제적인 유연성 확보에 집중했습니다. 역할 : 비즈니스 로직 변화를 예측하여 모듈화를 극대화하고, 인터페이스 기반으로 코드를 설계함으로써 향후 요구사항 변경 시에도 최소한의 수정으로 기능을 확장할 수 있는 코드를 제공했습니다.
