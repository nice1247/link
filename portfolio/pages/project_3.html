PL (Project Leader) 역할 및 품질 관리

프로젝트 전반의 백엔드 개발 및 데이터 모델링을 담당함과 동시에, 3인 개발팀의 PL 역할을 수행하며 프로젝트 성공을 이끌었습니다.

  • 코드 및 프로세스 통일화: 공용 모듈 개발을 주도하고, 프로젝트 전체의 변수명 및 서비스 프로세스 통일화를 진행하여 협업 효율성 및 코드 가독성을 극대화했습니다.
  • 일정 및 자원 관리: 업무 단위별 명확한 일정을 분배하고, 정기적인 코드 검토(Code Review)를 통해 개발 표준 준수와 품질 유지를 철저히 관리했습니다.

기술 스택

Spring Framework Java MySQL (MariaDB) Redis (세션/캐싱) GitLab

프레임워크 Spring Framework, Java
데이터베이스 MySQL (MariaDB)
핵심 인프라 Redis (세션 유지 및 캐싱 관리)

핵심 기여 및 기술적 난제 해결

1. 상이한 예약 모델 동시 구현 및 유연한 운영 시스템 구축

공통 모듈을 기반으로, 시설 대관 예약 등 상이한 비즈니스 요구사항을 충족하는 두 가지 예약 모델을 동시 구현했습니다.

[성과] 예약 유형별로 차별화된 설정 관리 시스템을 백엔드에 구축하여, 관리자가 코드를 수정하지 않고도 복잡한 예약 로직을 유연하게 운영할 수 있도록 지원했습니다.

2. 외부 환경 세션 유지 난제 해결 및 안정성 확보

네이버 앱 등 외부 앱 브라우저 환경에서 발생하는 간헐적인 세션 끊김 현상으로 예약이 실패하는 치명적인 문제를 해결했습니다.

[해결] Redis를 도입하여 기존 Tomcat 세션 대신 중앙 집중식 세션 관리를 수행하도록 구조를 변경했습니다.

[성과] 세션 안정성을 획기적으로 확보하여 모든 브라우저 환경에서 예약 프로세스의 무결성을 보장했습니다.

3. 예측 기반 요구사항 대비 및 코드 유연성 확보

잦은 변동과 피드백이 예상된 예약 시스템 개발 시, 선제적인 유연성 확보에 집중했습니다.

[역할] 비즈니스 로직 변화를 예측하여 모듈화를 극대화하고, 인터페이스 기반으로 코드를 설계하여 향후 요구사항 변경 시 최소한의 수정으로 기능을 확장할 수 있도록 대비했습니다.

주요 경험 및 역량

  • 기술적 난제 해결: 외부 앱 브라우저 환경에서 발생한 세션 문제를 Redis 도입으로 근본적으로 해결하며, 인프라 및 시스템 안정화 역량을 입증했습니다.
  • 리딩 능력 및 품질 관리: 3인 팀 PL로서 개발 표준 확립 및 코드 검토를 통해 고품질 결과물을 달성하는 프로젝트 관리 역량을 입증했습니다.
