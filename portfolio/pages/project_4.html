1. 프로젝트 명 : 이천호국원 예약 및 호출 시스템 구축 서비스 역할 : 웹, 앱, 키오스크를 통합한 예약 시스템 및 현장 사용자 호출(사이니지) 시스템 제공 참여 기간 : 2025년 09월 ~ 2025년 11월 URL : 이천호국원 홈페이지

2. 프로젝트 전반의 백엔드 아키텍처 설계 및 데이터 모델링을 담당했습니다. 웹, 앱, 키오스크 등 다중 채널 환경에서 예약 기능의 동일한 동작 및 무결성을 확보하는 데 주력했으며, 실시간 소켓 통신 및 커스텀 JWT를 도입하여 시스템의 성능과 보안을 최적화했습니다.

3. 프레임워크: Spring Framework, Java 데이터베이스: MySQL (MariaDB) 통신/보안: WebSocket, Custom JWT Token 웹/서버: Apache, Tomcat, JavaScript 개발 도구: Swagger (API 문서화)

4. - 웹, 앱, 키오스크 통합 예약 시스템 구축 [다중 채널 일관성 확보] 웹, 모바일 앱, 현장 키오스크 등 세 가지 상이한 채널에서 동작하는 예약 시스템의 백엔드 로직을 구현했습니다. 성과 : 예약, 취소, 조회 등 핵심 기능의 API 로직을 단일화된 구조로 설계하여, 협업 주체가 상이함에도 불구하고 모든 채널에서 동일하게 기능하도록 시스템의 일관성과 안정성을 보장했습니다. - 대용량 데이터 처리를 위한 커서 페이징 도입 [성능 최적화] 조회 시간이 중요한 예약 및 호출 리스트에서 오프셋(Offset) 방식 페이징 사용 시 발생하는 성능 저하 및 지연 현상을 해결하기 위해 커서 방식 페이징을 도입했습니다. 성과 : WHERE 절에 PK 또는 고유 인덱스 값을 활용하는 커서 기반 방식을 적용하여, 데이터베이스의 부하를 획기적으로 줄이고 리스트 로딩 속도를 -- 이상 개선했습니다. - 실시간 호출 시스템 구현 및 QR 인증 로직 개발 [소켓 통신 구현] 은행의 번호표 호출 시스템과 유사하게, 사용자(키오스크/앱)와 관리자 간의 실시간 호출 및 상태 업데이트를 위한 소켓 통신 모듈을 백엔드에 구축했습니다. [QR 인증 개발] 현장에서 예약자 식별을 위한 QR 코드 생성 및 검증 로직을 구현하고, 중복 사용 방지 및 현재 예약자만 구분하여 시스템의 정확성을 높였습니다. - 보안 및 협업 효율 강화를 위한 인증 시스템 구축 [커스텀 JWT 도입] 시스템의 보안을 강화하고 API 호출의 무결성을 확보하기 위해 Custom JWT (JSON Web Token) 방식을 설계 및 적용했습니다. 역할 : 인증 과정 및 토큰 검증 로직을 직접 구현하여, 관리자 등급별 권한 처리를 토큰 내에 포함시켜 API 호출 권한을 세밀하게 제어했습니다. [Swagger 최신화] 앱 개발팀과의 원활한 협업을 위해 Swagger 버전을 업그레이드하고, 모든 API 명세서를 최신 상태로 관리하여 API 변경사항에 대한 커뮤니케이션 비용을 절감했습니다. - PK 없는 데이터 처리를 위한 논리 모델링 [데이터 처리 난제 해결] PK(Primary Key) 값을 처리할 수 없는 고유 값이 없는(Non-PK) 데이터에 대한 등록 및 업데이트 처리를 위해 논리적인 모델링 및 처리 방식을 정의했습니다. 성과 : 클라이언트 요구사항을 만족시키며 시스템 무결성을 해치지 않으면서도 외부 시스템의 데이터 제약 조건을 수용할 수 있는 유연한 데이터 처리 아키텍처를 구축했습니다.

5. - 고성능 데이터 처리 : 대용량 환경에서 리스트 페이징의 성능 문제를 커서 방식으로 해결하며, 단순 기능 구현을 넘어 시스템 성능 최적화에 대한 깊이 있는 이해를 얻었습니다. - 통합 솔루션 경험 : 실시간 통신(소켓), 보안(JWT), 성능(커서 페이징) 등 다양한 핵심 백엔드 기술 요소를 통합하여 하나의 복합적인 시스템을 성공적으로 구축하는 역량을 입증했습니다. - 다중 플랫폼 API 무결성 확보 : 웹, 앱, 키오스크가 하나의 백엔드 API를 사용하는 환경에서 협업 부서와 관계없이 기능의 일관성을 유지하는 API 설계 및 관리 능력을 키웠습니다.
