프로젝트 개요

다중 채널에서 “같은 예약”이 돌아가야 하고, 현장에서는 “호출”이 끊기면 바로 민원이 되는 구조였습니다. 그래서 기능 나열이 아니라, 시스템 안정성을 좌우하는 축을 먼저 잡고 구현했습니다.

  • API 로직 단일화
    Consistency

    웹/앱/키오스크에서 공통으로 쓰는 도메인 로직을 중심에 두고 채널은 “입력/표현”으로 분리.

  • 커서 페이징 도입
    Performance

    오프셋 페이징의 누적 비용을 제거하고, 인덱스 기반 WHERE 조건으로 리스트 조회를 안정화.

  • WebSocket 호출/상태 전파
    Real-time

    호출/대기/완료 상태 변경을 즉시 전파하고, 사이니지 화면이 “항상 최신”이 되도록 구성.

  • Custom JWT로 권한/보안 강화
    Security

    관리자 등급/권한을 클레임으로 담아 호출 권한을 세밀하게 제어하고 API 무결성을 강화.

  • Non-PK 데이터 논리 모델링
    Data modeling

    고유키가 없는 데이터를 안전하게 식별/갱신할 수 있도록 논리 키/처리 규칙을 정의.

프로젝트를 통해 얻은 역량

  • 고성능 데이터 처리
    대용량 리스트 병목을 커서 방식으로 해결하며 최적화 관점을 강화.
  • 복합 시스템 통합
    실시간(WebSocket)·보안(JWT)·성능(페이징)을 묶어 안정적인 구조로 구현.
  • API 무결성 설계
    채널이 늘어도 동일한 결과가 나오도록 도메인 중심으로 정리.
