프로젝트 전반의 **백엔드 아키텍처 설계** 및 **데이터 모델링**을 담당했습니다. 다중 채널 환경에서 예약 기능의 **동일 동작 및 무결성 확보**에 주력했으며, 실시간 소켓 통신 및 커스텀 JWT를 도입하여 시스템의 성능과 보안을 최적화했습니다.

🛠️ 주요 기술 스택

Spring Framework Java MySQL (MariaDB) WebSocket Custom JWT Swagger

프레임워크 Spring Framework, Java
통신/보안 WebSocket, Custom JWT Token
개발 도구 Swagger (API 문서화)

🚀 핵심 기여 및 시스템 최적화

1. 웹/앱/키오스크 통합 예약 시스템 구축

세 가지 상이한 채널에서 동작하는 예약 시스템의 API 로직을 단일화된 구조로 설계했습니다.

[성과] 예약, 취소, 조회 등 핵심 API 로직을 단일화하여 모든 채널에서 **동일한 기능과 일관성**을 보장했습니다.

2. 대용량 데이터 처리를 위한 커서 페이징 도입

조회 시간이 중요한 예약 및 호출 리스트에서 오프셋 방식 페이징 사용 시 발생하는 성능 저하를 해결했습니다.

[방법] 커서 방식 페이징을 적용하여 WHERE 절에 PK 또는 고유 인덱스 값을 활용했습니다.

[성과] 데이터베이스 부하를 획기적으로 줄이고 리스트 로딩 속도를 **크게 개선**하며 고성능 조회를 실현했습니다.

3. 실시간 호출 시스템 (WebSocket) 및 QR 인증 로직 개발

은행 번호표 호출 시스템과 유사한 실시간 호출 및 상태 업데이트를 위한 소켓 통신 모듈을 구축했습니다. 현장 예약자 식별을 위한 QR 생성/검증 로직도 함께 구현했습니다.

4. 보안 및 협업 효율 강화를 위한 인증 시스템 구축

시스템 보안 강화 및 API 무결성 확보를 위해 Custom JWT (JSON Web Token) 방식을 직접 설계하고 적용했습니다.

[역할] 인증 과정 및 토큰 검증 로직을 구현하여, 관리자 등급별 권한 처리를 토큰 내에 포함시켜 API 호출 권한을 **세밀하게 제어**했습니다. Swagger 최신화로 앱 개발팀과의 협업 효율을 높였습니다.

5. PK 없는 데이터 처리를 위한 논리 모델링

PK(Primary Key) 값을 처리할 수 없는 고유 값이 없는(Non-PK) 데이터에 대한 등록 및 업데이트 처리를 위해 논리적인 모델링 및 처리 방식을 정의했습니다.

[성과] 시스템 무결성을 해치지 않으면서도 외부 시스템의 데이터 제약 조건을 수용할 수 있는 유연한 데이터 처리 아키텍처를 구축했습니다.

🌟 프로젝트를 통한 주요 경험 및 역량

  • **고성능 데이터 처리:** 대용량 환경에서 리스트 페이징의 성능 문제를 커서 방식으로 해결하며, 시스템 성능 최적화에 대한 깊이 있는 이해를 얻었습니다.
  • **통합 솔루션 경험:** 실시간 통신(소켓), 보안(JWT), 성능(커서 페이징) 등 핵심 백엔드 기술 요소를 통합하여 복합적인 시스템을 성공적으로 구축하는 역량을 입증했습니다.
  • **다중 플랫폼 API 무결성 확보:** 웹, 앱, 키오스크 환경에서 협업 부서와 관계없이 기능의 일관성을 유지하는 **API 설계 및 관리 능력**을 키웠습니다.
